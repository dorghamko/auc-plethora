Element Details

Frame

The root node of the document. Frame must contain a Properties element, a NonVisualElements element, a Layout element, and a Components element in that order. id is always fixed to "this" to maintain a reference to the generated class object which will make code generation easier.

Declared Attributes

  • #FIXED CDATA id = this

Properties

SimpleProperty

The Simple Property does not require an object to be created first, and it can only take one value. Example: foo.setHeight(10.4f);

Element

Element defines a class object that can be used either as a visual element in a component such as a sphere or as a non-visual element such as appearance. The value of the id attribute is unique and represents the element's variable name. The type attribute defines the class type of the element.

EventHandlers

EventHandlers represents a group of event handlers that are defined on a component or a container.

Element Content Model

(EventHandler)*

Referenced by

EventHandler

EventHandler defines an event handler that handles an event on a component or a container. The value of the event attribute represents the name of the event which is the name of the overriden function in the inner annonymous class. The value of the listener attribute represents the interface to be implemented or the abstract class to be extended by the inner annonymous class. The value of the parameter attribute represents the class type of the passed parameter. The value of the handler attribute represents the name of the function to which the event is propagated; this value must be unique since it is a function name.
